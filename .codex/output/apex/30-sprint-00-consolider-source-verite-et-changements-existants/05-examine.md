## Adversarial Review

Four independent reviews covered payment integrity, business logic, maintainability
and the Next.js/React UI. The following findings were confirmed and resolved:

| Severity | Finding | Resolution |
| --- | --- | --- |
| Critical | A retry could replace the Fapshi identifiers of an earlier checkout. | An existing transaction is now returned unchanged; a player must cancel before creating a new attempt. |
| Critical | Reconciliation could settle a payment without a validated amount. | Webhook and worker both require the exact expected positive integer amount. |
| High | Webhook success with a missing amount could be accepted. | Such events are audited and rejected without promoting the registration. |
| High | Cancellation could race with late provider settlement. | Provider expiry is attempted first and guarded database transitions preserve a successful settlement. |
| High | Concurrent payment/webhook transitions were not sufficiently protected. | Serializable transactions, idempotency and guarded `updateMany` transitions were added. |
| High | Login and registration accepted arbitrary `next` navigation. | A shared `safeInternalRedirect` rejects external and malformed destinations. |
| Medium | A late wallet fetch could overwrite an explicit Fapshi choice. | The drawer tracks an explicit method choice before applying defaults. |
| Medium | Account changes could render stale profile/history data. | Effects now use an abort signal and active request guard keyed by user identity. |
| Medium | Payment success navigation contained nested interactive elements. | The UI uses the button render API to produce a single link control. |
| Medium | Admin session creation defaulted to public visibility. | API and form defaults are now private. |
| Medium | Health polling continued too aggressively in background tabs. | Visibility-aware polling, cancellation and exponential backoff were added. |

The report artifact generated by Playwright remains in scope because the user explicitly
requested that all existing changes be committed. No source code relies on that artifact.

## Residual Test Quality Note

The existing web page tests inspect source text because the repository has no component
test renderer. The most security-sensitive redirect logic now has direct unit tests, and
the public catalogue flow is mandatory in Playwright. A future sprint can introduce a
component renderer if broader UI behavior coverage is required.
