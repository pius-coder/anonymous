## Resume executif

Le depot `session-jeu` est un monorepo **pnpm + Turbo** en reconstruction progressive.
La fondation v0.1 n'est pas un MVP jetable : elle reconstitue une plateforme multijoueur ambitieuse
(auth, paiements, live, admin, scoring, mini-jeux) avec des frontieres de couche explicites.

**Position actuelle** : socle technique solide, UI large en avance sur le branchement backend.

## Decisions architecturales respectees

Confirmes dans le code et la doc :

- Timer et jobs ne demarrent jamais une partie active sans commande admin.
- Scores provisoires en DB ; pas d'API de publication encore.
- Parcours admin / joueur / observer separes au niveau routes et layouts.
- Serveur autoritaire pour les positions; autorite round/deadline presente mais flux normal incomplet.
- Contrats Protobuf comme cible reseau (REST encore dominant cote API).

## Ecarts majeurs vs cible documentaire

1. **UI non cablee** — Cinq modules (`ui-data.ts`, `player-data.ts`, `finance-data.ts`,
   `observer-data.ts`, `support-data.ts`) alimentent 32 fichiers source au lieu des endpoints existants.
2. **ConnectRPC incomplet** — Seuls `IdentityService`, `RoundService`, `RealtimeAccessService`
   sont enregistres dans `apps/api/src/rpc/routes.ts`.
3. **Scoring** — Modeles Prisma + transitions domaine ; zero route API/RPC.
4. **Mini-jeux** — Zero runtime ; room Colyseus = social + orchestration round partiellement cablee.
5. **BullMQ** — Dependance declaree, aucun Worker/Queue en runtime.
6. **Seed** — `packages/db/prisma/seed.ts` vide ; setup manuel requis.
7. **Integration metier** — SEQ-00 fournit CI, DB jetable et smokes multi-service; les parcours metier
   authentifies, la reconnexion et les claims concurrents restent a prouver.
