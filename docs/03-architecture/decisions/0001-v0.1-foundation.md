# ADR 0001 - Fondation v0.1

## Decision

Repartir d'une documentation cible et d'une architecture par cas d'usage, sans correction progressive du legacy `/live`.

## Raisons

- Les parcours admin et joueur sont confondus.
- Le live actuel melange preparation, manche, mini-jeu, social, observation et resultats.
- La cible exige Protobuf, separation des acteurs, et verification/publication explicite.

## Consequences

- Le legacy est audite et classe.
- Les suppressions code attendent validation du manifeste.
- Les futures features commencent par use case et contrats, pas par composants.
