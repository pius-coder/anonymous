## Checklist Par User Story

Avant de marquer une user story UI prete:

- [ ] Acteur clairement identifie.
- [ ] Route ou surface indiquee.
- [ ] Etat loading/empty/error/denied/stale/reconnect couvert si pertinent.
- [ ] CTA principal et CTA interdit documentes.
- [ ] Donnees visibles et donnees interdites listees.
- [ ] Contrat/projection attendue listee.
- [ ] Test UI ou E2E attendu defini.
